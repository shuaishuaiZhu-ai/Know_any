### 文档概述

本文档基于 UCX（Unified Communication X）源代码分析，针对国产 AI 芯片设计提出详细的硬件特性需求。UCX 作为高性能通信框架，广泛应用于分布式 AI 训练和推理系统，国产 AI 芯片要实现与 UCX 的完全兼容，需要在硬件层面提供相应的支持能力。本文基于ucx 版本 (master 分支 commit b13cf9b5141500ed2fc40a7b2fdef341510aebab )。

**分析基础**：
- UCX 源码版本：基于 `ucx` 仓库
- 重点分析模块：UCT 传输层、UCS 服务层、GPU/RDMA 支持
- 目标芯片类型：训练加速芯片、推理加速芯片、AI 服务器芯片
- 适用范围: 国产训练/推理 AI 芯片设计
- 参考标准: UCX 1.12+, CUDA 11.8+, Linux Kernel 5.10+

### 二、GPU 内存管理硬件特性需求

### 2.1 内存分配与释放

**硬件需求**：

```
<span></span>// 参考：ucx/src/uct/cuda/cuda_copy/cuda_copy_md.c

// 1. 标准设备内存分配
CUresult cuMemAlloc(CUdeviceptr *dptr, size_t bytesize);
CUresult cuMemFree(CUdeviceptr dptr);

// 2. 统一内存（Managed Memory）
CUresult cuMemAllocManaged(CUdeviceptr *dptr, size_t bytesize,
                           unsigned int flags);

// 3. 主机内存注册（Pinned Memory）
CUresult cuMemHostRegister(void *p, size_t bytesize, unsigned int Flags);
CUresult cuMemHostUnregister(void *p);

```

**国产芯片必需实现**：

1, **设备内存分配器**：
- 支持大页内存分配（>=2MB）
- 内存对齐要求：至少 256 字节对齐
- 支持异步内存分配（避免同步开销）
2, **统一内存架构**：

```
<span></span>统一内存特性要求

   +-----------------------+
   | CPU 可访问             |
   | GPU 可访问             |
   | 自动页迁移             |  ← 硬件支持
   | 按需调页（On-Demand）  |
   +-----------------------+

   性能指标：
   - CPU→GPU 迁移：<100μs (4KB 页)
   - GPU→CPU 迁移：<100μs
   - 并发迁移支持：>=16 个页面同时迁移

```

3, **主机内存锁页（Pinning）**：
- 硬件 DMA 引擎直接访问锁页内存
- 支持物理地址连续性保证
- 锁页内存容量：建议 >=物理内存的 50%

### 3.2 P2P 内存访问

**硬件需求**（高优先级）：

**国产芯片硬件架构要求**：

1, **片间互连拓扑**：

```
<span></span>推荐拓扑：全连接 Mesh（8 GPU 示例）

   GPU0 - GPU1 - GPU2 - GPU3
    |  ×   |  ×   |  ×   |
   GPU4 - GPU5 - GPU6 - GPU7

   每对 GPU 间：
   - 直连带宽：>=300 GB/s
   - 双向带宽：>=600 GB/s
   - 跳数：<=2（通过中间 GPU）

```

2, **P2P DMA 引擎**：
- 独立 DMA 通道：>=8 个/GPU
- 并发 DMA 传输：>=16 个
- DMA 描述符队列：>=1024 条目
- 支持分散-聚集（Scatter-Gather）DMA
3, **地址空间管理**：

```
<span></span>统一虚拟地址空间（GVA - Global Virtual Addressing）

   GPU 0 地址空间          GPU 1 地址空间
   +------------------+    +------------------+
   | 0x0000_0000_0000 |    | 0x0001_0000_0000 |
   |   本地设备内存    |    |   本地设备内存    |
   +------------------+    +------------------+
   | 0x0002_0000_0000 |    | 0x0003_0000_0000 |
   |   映射远端 GPU 1  |    |   映射远端 GPU 0  |
   +------------------+    +------------------+

   硬件要求：
   - 64 位地址空间
   - MMU 支持远端地址转换
   - TLB 大小：>=8192 条目

```

### 四、RDMA 硬件支持需求

### 4.1 GPUDirect RDMA 能力

**硬件需求**（核心竞争力）：

```
<span></span>GPUDirect RDMA 架构（对标 NVIDIA GPUDirect）

+------------------+         +------------------+
|   GPU 设备内存    |         |   远端 GPU       |
|   物理地址可暴露  |         |   （网络对端）   |
+--------+---------+         +--------+---------+
         |                            ^
         |                            |
         | PCIe/NVLink               | 网络传输
         v                            |
+--------+---------+         +--------+---------+
|   NIC（RDMA）    |         |   NIC（RDMA）    |
|   - DMA 引擎     | <-----> |   - DMA 引擎     |
|   - GPU 地址转换 |  IB/RoCE |   - GPU 地址转换 |
+------------------+         +------------------+

关键硬件特性：
1. GPU BAR（Base Address Register）大小
   - 必需：>=设备内存总量
   - 推荐：全尺寸 BAR（如 64GB GPU 需 64GB BAR）

2. PCIe 配置
   - PCIe Gen4 x16 或更高
   - 支持 PCIe ATS（Address Translation Services）
   - 支持 PCIe ACS（Access Control Services）

3. IOMMU 兼容性
   - 支持 IOMMU pass-through
   - 支持 PASID（Process Address Space ID）

```

**国产芯片实现要点**：

1, **GPU 内存物理地址暴露**：

```
<span></span>// 驱动需提供的接口
   typedef struct {
       uint64_t physical_addr;    // GPU 物理地址
       size_t size;               // 区域大小
       uint32_t bar_id;           // PCIe BAR 编号
       uint64_t bar_offset;       // BAR 内偏移
   } gpu_rdma_memory_info_t;

   int get_gpu_rdma_memory_info(void *gpu_ptr,
                                gpu_rdma_memory_info_t *info);

```

2, **RDMA 注册内存**：

```
<span></span>// 与 RDMA 网卡驱动集成
   int register_gpu_memory_for_rdma(
       void *gpu_ptr,           // GPU 设备指针
       size_t size,             // 大小
       uint64_t *physical_addr, // 返回物理地址
       uint32_t *lkey,          // 本地密钥
       uint32_t *rkey           // 远程密钥
   );

```

3, **性能指标要求**：

```
<span></span>GPUDirect RDMA 性能基准

   延迟：
   - 单向延迟（GPU→远端GPU）：<5μs
   - 双向延迟（GPU↔远端GPU）：<10μs

   带宽：
   - 单向带宽：>=100 GB/s（200Gbps IB HDR）
   - 双向带宽：>=200 GB/s
   - 多 GPU 聚合：线性扩展

   CPU 占用：
   - RDMA 传输期间 CPU 占用：<5%
   - 零拷贝验证：是（无 CPU 中转）

```

```
+------------+         +-------------+          +------------+

| 导出为 Fabric       | - 低延迟      |          | 导入为本地 |
+------------+         | - 高带宽      |          +------------+

硬件特性：
1. Fabric 内存分配
   - 可被远端节点直接访问
   - 物理内存连续性
   - 支持原子操作

2. Fabric 地址转换
   - 全局地址空间
   - 节点 ID + 本地地址
   - 硬件路由单元
```

### 7.2 系统级原子一致性

**硬件缓存一致性协议**：

```
<span></span>多 GPU 缓存一致性（类似 MOESI 协议）

状态转换：
M (Modified)    ──┐
E (Exclusive)   ──┼──> 本地修改、其他 GPU 无效

   - 支持 >=8 GPU 的一致性维护
```

```
typedef struct uct_md_ops {
```

**示例：国产芯片 MD 实现框架**：

```
<span></span>// domestic_chip_md.c

static ucs_status_t domestic_chip_md_query(uct_md_h md,
                                           uct_md_attr_v2_t *attr)
{
    attr->flags = UCT_MD_FLAG_REG | UCT_MD_FLAG_ALLOC;

    // 支持的内存类型
    attr->reg_mem_types = UCS_BIT(UCS_MEMORY_TYPE_HOST) |
                          UCS_BIT(UCS_MEMORY_TYPE_DOMESTIC_DEVICE) |
                          UCS_BIT(UCS_MEMORY_TYPE_DOMESTIC_MANAGED);

    attr->alloc_mem_types = UCS_BIT(UCS_MEMORY_TYPE_DOMESTIC_DEVICE) |
                            UCS_BIT(UCS_MEMORY_TYPE_DOMESTIC_MANAGED);

    attr->access_mem_types = attr->reg_mem_types;
    attr->detect_mem_types = attr->reg_mem_types;

    // dmabuf 支持
    attr->dmabuf_mem_types = domestic_chip_dmabuf_supported() ?
                             UCS_BIT(UCS_MEMORY_TYPE_DOMESTIC_DEVICE) : 0;

    return UCS_OK;
}

static ucs_status_t domestic_chip_mem_reg(uct_md_h md, void *address,
                                          size_t length,
                                          const uct_md_mem_reg_params_t *params,
                                          uct_mem_h *memh_p)
{
    domestic_mem_type_t mem_type;

    // 检测内存类型
    mem_type = domestic_mem_get_type(address);

    if (mem_type == DOMESTIC_MEM_TYPE_HOST) {
        // 主机内存锁页
        return domestic_mem_host_register(address, length, memh_p);
    } else if (mem_type == DOMESTIC_MEM_TYPE_DEVICE) {
        // GPU 内存已经注册，返回句柄
        *memh_p = address;
        return UCS_OK;
    }

    return UCS_ERR_INVALID_PARAM;
}

static uct_md_ops_t domestic_chip_md_ops = {
    .query         = domestic_chip_md_query,
    .mem_reg       = domestic_chip_mem_reg,
    .mem_dereg     = domestic_chip_mem_dereg,
    .mem_alloc     = domestic_chip_mem_alloc,
    .mem_free      = domestic_chip_mem_free,
    .mkey_pack     = domestic_chip_mkey_pack,
};

```
