### 1. 概述

### 1.1 NVSHMEM简介

NVSHMEM是NVIDIA开发的一个高性能并行编程接口库,为多GPU和多节点环境提供共享内存编程模型。它扩展了OpenSHMEM标准,专门针对NVIDIA GPU进行了优化,使GPU能够直接访问远程GPU的内存,从而实现高效的点对点和集合通信。本文档基于nvshmem版本`2.4.1-3`。

### 1.2 核心特性
- 对称堆内存管理: 所有PE(Processing Element)具有相同大小的对称堆
- 多传输层支持: P2P(NVLink/PCIe)、InfiniBand RC、UCX
- GPU端通信原语: 直接从CUDA kernel发起通信操作
- 代理机制: 支持CPU辅助的异步通信
- 团队(Team)支持: 支持通信子集和层次化通信
- 集合操作: Barrier、Broadcast、Reduction、AllToAll等
### 1.3 4+1视图模型说明

本分析采用Philippe Kruchten提出的4+1架构视图模型:
- 逻辑视图: 系统功能分解和模块关系
- 过程视图: 运行时进程、线程和通信流程
- 开发视图: 代码组织和模块划分
- 物理视图: 系统在硬件上的部署
- 场景视图: 关键用例和交互序列

**三种传输实现**:

### P2P传输 (ID=0)
- 适用场景: 单节点或NVSwitch互联
- 技术: CUDA IPC (cudaIpcGetMemHandle)
- 延迟: <1μs (NVLink)
- 带宽: 600GB/s (NVLink 4.0)
- 限制: 仅限同一物理主机
### IBRC传输 (ID=1)
- 适用场景: 多节点InfiniBand集群
- 技术: IB Verbs RC (Reliable Connection)
- 延迟: ~1-2μs
- 带宽: 200Gb/s (HDR IB)
- 特性: GPUDirect RDMA支持
### UCX传输 (ID=2)
- 适用场景: 异构网络环境
- 技术: UCX框架 (统一通信X)
- 后端: IB/RoCE/TCP/shared memory
- 优势: 自动选择最优传输路径
**传输选择矩阵**:

```
目标PE    同GPU  同节点  跨节点
─────────────────────────────
传输      P2P    P2P    IBRC/UCX
延迟      最低   低     中
GPUDirect  ✓      ✓      ✓(with GDR)
```

### 2.2.5 API层设计

**API分类**:

```
1. RMA操作 (Remote Memory Access):
   - `nvshmem_put()` / `nvshmem_get()`
   - `nvshmem_put_nbi()` / `nvshmem_get_nbi()` (非阻塞)
   - `nvshmemx_put_block()` (多线程协作)

2. AMO操作 (Atomic Memory Operations):
   - `nvshmem_atomic_add()`
   - `nvshmem_atomic_fetch_and()`
   - `nvshmem_atomic_compare_swap()`

3. 同步原语:
   - `nvshmem_fence()` - 本地内存序
   - `nvshmem_quiet()` - 等待所有操作完成
   - `nvshmem_barrier_all()` - 全局barrier

4. 集合通信:
   - `nvshmem_broadcast()` - 广播
   - `nvshmem_reduce()` - 归约
   - `nvshmem_alltoall()` - 全到全

5. 信号操作:
   - `nvshmem_signal_op()` - 远程信号
   - `nvshmem_wait_until()` - 本地等待
```

### 3.3 集合操作: Barrier

**算法**: Dissemination Barrier

```
轮次k: 每个PE与PE_id XOR (1 << k)交换信号
总轮次: ceil(log2(npes))
```

**算法原理**:

```
4 PEs示例:
PE0: 0000  PE1: 0001  PE2: 0010  PE3: 0011

轮次0 (XOR 0001):
  PE0 ↔ PE1
  PE2 ↔ PE3

轮次1 (XOR 0010):
  PE0 ↔ PE2
  PE1 ↔ PE3

总共2轮 = log2(4)
```

**GPU实现**:

```
__device__ void nvshmem_barrier_all() {
    int log2_npes = 0;
    while ((1 << log2_npes) < npes) log2_npes++;

    for (int i = 0; i < log2_npes; i++) {
        int peer = mype ^ (1 << i);

        // 发送信号给peer
        nvshmem_signal_op(&psync[offset], value, SET, peer);

        // 等待peer的信号
        nvshmem_wait_until(&psync[my_offset], CMP_EQ, expected);

        expected++; // 下一轮期望值
    }
}
```

**性能特点**:
- 时间复杂度: O(log N)
- 消息数: N log N (总)
- 最大距离: N/2 (跨节点可能)
- 优化: 分层barrier (先节点内,再节点间)

nvshmem_float_broadcast(NVSHMEM_TEAM_WORLD, A, A, M*K, 0);
nvshmem_float_broadcast(NVSHMEM_TEAM_WORLD, B, B, K*N, 0);

// 步骤4: 本地计算
int local_rows = M / npes;
int row_offset = mype * local_rows;
matmul_kernel<<<...>>>(A + row_offset*K, B, C_local, ...);

// 步骤5: PE0收集结果
nvshmem_barrier_all();
if (mype == 0) {
    for (int pe = 1; pe < npes; pe++) {
        nvshmem_float_get(C_full + pe*local_rows*N,
                         C_local, local_rows*N, pe);
    }
}
```

**交互序列**:

```
1. init阶段:
   所有PE → nvshmem_init() → 分配对称堆

2. 数据分发:
   PE0 → nvshmem_broadcast(matrix_A) → 所有PE
   PE0 → nvshmem_broadcast(matrix_B) → 所有PE

3. 本地计算:
   各PE → 计算本地分块 → 存入对称堆

4. 结果收集:
   PE0 → nvshmem_get(结果块, pe_i) → 从其他PE获取

5. 同步:
   所有PE → nvshmem_barrier_all() → 确保完成
```

### 6.2 用例: Ring AllReduce

**场景**: 高效的梯度聚合算法。

**算法详解**:

```
假设4个PE, 每个PE朄64个元素，总计256元素

Reduce-Scatter阶段 (3轮):
┌─────────────────────────────────────────────────┐
│ PE0    PE1    PE2    PE3                         │
│ ┌────┐ ┌────┐ ┌────┐ ┌────┐                │
│ │C0  │ │C1  │ │C2  │ │C3  │ 初始状态      │
│ ├────┤ ├────┤ ├────┤ ├────┤ (每个64元素) │
│ │C1  │ │C2  │ │C3  │ │C0  │                │
│ ├────┤ ├────┤ ├────┤ ├────┤                │
│ │C2  │ │C3  │ │C0  │ │C1  │                │
│ ├────┤ ├────┤ ├────┤ ├────┤                │
│ │C3  │ │C0  │ │C1  │ │C2  │                │
│ └────┘ └────┘ └────┘ └────┘                │
└─────────────────────────────────────────────────┘

轮次11: 每个PE发送chunk[3]到下一PE，接收并累加chunk[2]
   PE0: C3 → PE1,  PE3.C2 → PE0.C2 += C2
   结果：PE0的C2包含2个PE的和

轮次12: 继续传递和累加
   PE0: C2 → PE1,  PE3.C1 → PE0.C1 += C1
   结果：PE0的C1包含3个PE的和

轮次13: 最后一轮
   PE0: C1 → PE1,  PE3.C0 → PE0.C0 += C0
   结果：PE0的C0包含所有4PE的和

AllGather阶段 (3轮):
   每个PE将其已归约的chunk传递，直到所有PE拥有全部数据
```

**步骤**:

```
1. Reduce-Scatter阶段:
   for i in range(npes-1):
       send_pe = (mype + 1) % npes
       recv_pe = (mype - 1 + npes) % npes

       nvshmem_put(send_chunk, send_pe)
       nvshmem_fence()
       nvshmem_wait_until(recv_flag)

       本地归约(recv_chunk)

2. AllGather阶段:
   for i in range(npes-1):
       nvshmem_put(完整chunk, next_pe)
       nvshmem_wait_until(recv_flag)
```

**性能分析**:

```
数据量: N元素
PE数量: P

通信量: 2 * (P-1) * (N/P) = 2N * (P-1)/P ≈ 2N
对比朴素AllReduce: P * N (大P时非常高)

时间: 2 * (P-1) * (α + β*N/P)
  α: 延迟
  β: 逆带宽
```

### 8.2.1 多线程协作传输

**场景**: 利用多个线程并行传输大块数据

```
#define THREADS_PER_BLOCK 1024

__global__ void set_and_shift_kernel(float *send_data, float *recv_data,
                                     int num_elems, int mype, int npes) {
    int thread_idx = blockIdx.x * blockDim.x + threadIdx.x;

    // 1. 初始化发送数据
    if (thread_idx < num_elems)
        send_data[thread_idx] = mype;

    int peer = (mype + 1) % npes;

    // 2. Block Put：整个block协作传输
    int block_offset = blockIdx.x * blockDim.x;
    nvshmemx_float_put_block(
        recv_data + block_offset,           // 目标地址
        send_data + block_offset,           // 源地址
        min(blockDim.x, num_elems - block_offset),  // 元素数量
        peer                                // 目标PE
    );
    /*
     * 优势对比:
     * - nvshmem_float_p(): 每个元素一个RMA，IB下效率低
     * - nvshmem_float_put(): P2P时无法利用多线程
     * - nvshmemx_float_put_block(): 两者兼顾
     */
}

int main(int c, char *v[]) {
    int mype, npes, mype_node;
    float *send_data, *recv_data;
    int num_elems = 8192;
    int num_blocks;

    nvshmem_init();
    mype = nvshmem_my_pe();
    npes = nvshmem_n_pes();
    mype_node = nvshmem_team_my_pe(NVSHMEMX_TEAM_NODE);

    cudaSetDevice(mype_node);
    send_data = (float *)nvshmem_malloc(sizeof(float) * num_elems);
    recv_data = (float *)nvshmem_malloc(sizeof(float) * num_elems);

    num_blocks = num_elems / THREADS_PER_BLOCK;
    set_and_shift_kernel<<<num_blocks, THREADS_PER_BLOCK>>>(
        send_data, recv_data, num_elems, mype, npes);

    cudaDeviceSynchronize();

    // 数据验证
    float *host = new float[num_elems];
    cudaMemcpy(host, recv_data, num_elems * sizeof(float),
               cudaMemcpyDefault);
    int ref = (mype - 1 + npes) % npes;
    for (int i = 0; i < num_elems; ++i) {
        if (host[i] != ref) {
            printf("Error at %d of rank %d: %f\n", i, mype, host[i]);
            break;
        }
    }

    nvshmem_free(send_data);
    nvshmem_free(recv_data);
    nvshmem_finalize();
    return 0;
}
```

**性能优化要点**:
- 使用*_block变体API利用多线程
- 块内所有线程必须用相同参数调用
- 适用于P2P和IB传输

### 8.6.2 深度学习梯度AllReduce

```
// Ring AllReduce实现
__global__ void gradient_allreduce(float *gradients, size_t n_params,
                                   int mype, int npes) {
    // 1. Reduce-Scatter阶段
    size_t chunk_size = n_params / npes;
    for (int step = 0; step < npes - 1; step++) {
        int send_chunk = (mype - step + npes) % npes;
        int recv_chunk = (mype - step - 1 + npes) % npes;
        int send_pe = (mype + 1) % npes;
        int recv_pe = (mype - 1 + npes) % npes;

        // 发送chunk
        nvshmem_float_put(gradients + send_chunk * chunk_size,
                         gradients + send_chunk * chunk_size,
                         chunk_size, send_pe);
        nvshmem_fence();

        // 等待接收
        nvshmem_barrier_all();

        // 归约接收到的chunk
        for (int i = threadIdx.x; i < chunk_size; i += blockDim.x) {
            size_t idx = recv_chunk * chunk_size + i;
            // 本地已有值 + 接收到的值
        }
        nvshmem_barrier_all();
    }

    // 2. AllGather阶段
    for (int step = 0; step < npes - 1; step++) {
        int send_chunk = (mype + 1 - step + npes) % npes;
        int send_pe = (mype + 1) % npes;

        nvshmem_float_put(gradients + send_chunk * chunk_size,
                         gradients + send_chunk * chunk_size,
                         chunk_size, send_pe);
        nvshmem_fence();
        nvshmem_barrier_all();
    }
}

### 8.8.1 性能优化
1. 选择合适的API:
1. 单元素: `*_p()`
1. 小批量: `*_put()` / `*_get()`
1. 大批量: `*_put_nbi()` (非阻塞)
1. 多线程: `*_block()` 变体
1. 内存访问:

```
<span class="c1">// Good: 合并访问

<span class="c1">// Bad: 分散访问
</span><span class="c1"></span><span class="k">for</span> <span class="p">(</span><span class="kt">int</span> <span class="n">i</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span> <span class="n">i</span> <span class="o"><</span> <span class="n">N</span><span class="p">;</span> <span class="n">i</span><span class="o">++</span><span class="p">)</span>
    <span class="n">nvshmem_float_p</span><span class="p">(</span><span class="o">&</span><span class="n">dest</span><span class="p">[</span><span class="n">i</span><span class="p">],</span> <span class="n">src</span><span class="p">[</span><span class="n">i</span><span class="p">],</span> <span class="n">pe</span><span class="p">);</span>

### 8.8.2 正确性保证

 1，**内存一致性**:

```
nvshmem_put(...);
nvshmem_fence();
// 确保Put可见
nvshmem_signal_op();
// 再通知对方

```

**2，同步模式**:
1. `barrier_all()`: 全局同步
1. `fence()`: 本PE的Put/Get完成
1. `quiet()`: 本PE所有操作完成
1. 避免死锁:

```
 // 错误：循环依赖

 nvshmem_int_p(&target, value, peer);

nvshmem_wait_until(&flag, CMP_EQ, 1);
