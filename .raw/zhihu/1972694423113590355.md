## 1. 基础示例 - Ring Shift

### 1.1 功能描述

最简单的NVSHMEM应用，演示点对点通信：每个PE将自己的ID发送给环形拓扑中的下一个PE。

### 1.2 源代码 (`dev-guide-ring.cu`)

```
<span></span><span class="cp">#include</span> <span class="cpf"><stdio.h></span><span class="cp"></span>
<span class="cp">#include</span> <span class="cpf"><cuda.h></span><span class="cp"></span>
<span class="cp">#include</span> <span class="cpf"><nvshmem.h></span><span class="cp"></span>
<span class="cp">#include</span> <span class="cpf"><nvshmemx.h></span><span class="cp"></span>

<span class="c1">// Kernel: 简单的环形移位</span>
<span class="n">__global__</span> <span class="kt">void</span> <span class="nf">simple_shift</span><span class="p">(</span><span class="kt">int</span> <span class="o">*</span><span class="n">destination</span><span class="p">)</span> <span class="p">{</span>
    <span class="kt">int</span> <span class="n">mype</span> <span class="o">=</span> <span class="n">nvshmem_my_pe</span><span class="p">();</span>           <span class="c1">// 获取当前PE的ID</span>
    <span class="kt">int</span> <span class="n">npes</span> <span class="o">=</span> <span class="n">nvshmem_n_pes</span><span class="p">();</span>           <span class="c1">// 获取总PE数量</span>
    <span class="kt">int</span> <span class="n">peer</span> <span class="o">=</span> <span class="p">(</span><span class="n">mype</span> <span class="o">+</span> <span class="mi">1</span><span class="p">)</span> <span class="o">%</span> <span class="n">npes</span><span class="p">;</span>         <span class="c1">// 环形拓扑：下一个PE</span>

    <span class="c1">// 将本PE的ID发送到下一个PE</span>
    <span class="n">nvshmem_int_p</span><span class="p">(</span><span class="n">destination</span><span class="p">,</span> <span class="n">mype</span><span class="p">,</span> <span class="n">peer</span><span class="p">);</span>

<span class="kt">int</span> <span class="nf">main</span><span class="p">(</span><span class="kt">void</span><span class="p">)</span> <span class="p">{</span>

    <span class="c1">// 4. 启动kernel</span>

    <span class="n">printf</span><span class="p">(</span><span class="s">"%d: received message %d</span><span class="se">\n</span><span class="s">"</span><span class="p">,</span> <span class="n">nvshmem_my_pe</span><span class="p">(),</span> <span class="n">msg</span><span class="p">);</span>

    <span class="n">nvshmem_finalize</span><span class="p">();</span>
    <span class="k">return</span> <span class="mi">0</span><span class="p">;</span>
<span class="p">}</span>

```

### 1.3 关键API说明

| API | 功能 | 返回值 |
| --- | --- | --- |
| nvshmem_init() | 初始化NVSHMEM运行时 | void |
| nvshmem_my_pe() | 获取当前PE的rank | int |
| nvshmem_n_pes() | 获取总PE数量 | int |
| nvshmem_int_p(dest, value, pe) | 单元素Put操作 | void |
| nvshmem_malloc(size) | 分配对称堆内存 | void* |
| nvshmemx_barrier_all_on_stream(stream) | 流上的全局barrier | void |
| nvshmem_finalize() | 清理NVSHMEM运行时 | void |

### 1.4 执行流程

```
<span></span>PE 0                PE 1                PE 2
  │                   │                   │
  ├─ simple_shift     ├─ simple_shift     ├─ simple_shift
  │  (send 0→PE1)     │  (send 1→PE2)     │  (send 2→PE0)
  │                   │                   │
  └─ barrier ─────────┴─ barrier ─────────┴─ barrier
         │                   │                   │
      recv=2              recv=0              recv=1

```

### 2. 高性能批量传输 - Block Put

### 2.1 功能描述

演示使用Block级别的协作Put操作，充分利用多线程并行来提高带宽利用率。每个PE将一个浮点数组发送给下一个PE。

### 2.2 核心代码 (`put-block.cu`)

```
<span class="cp">#include</span> <span class="cpf"><assert.h></span><span class="cp"></span>
<span class="cp">#include</span> <span class="cpf">"nvshmem.h"</span><span class="cp"></span>

<span class="cp">#define THREADS_PER_BLOCK 1024</span>
```

### 2.3 性能对比

| 方法 | API | 并行度 | P2P吞吐 | RDMA吞吐 |
| --- | --- | --- | --- | --- |
| 单线程 | nvshmem_float_p() | 1线程 | 1x | 1x |
| 单线程Put | nvshmem_float_put() | 1线程 | 1x | 1x (多消息) |
| Block Put | nvshmemx_float_put_block() | Block线程 | 5-8x | 10-20x |

**关键优势**:
- P2P场景: 多线程并行拷贝，充分利用带宽
- RDMA场景: 合并为单个大消息，减少网络开销
- 编程简单: 所有线程调用相同参数的API
### 2.4 内存访问模式

```
<span></span>Block 0 (1024 threads)              Block 1 (1024 threads)
   │                                    │
   ├─ thread 0: offset 0                ├─ thread 0: offset 1024
   ├─ thread 1: offset 1                ├─ thread 1: offset 1025
   ├─ thread 2: offset 2                ├─ thread 2: offset 1026
   │    ...                             │    ...
   └─ thread 1023: offset 1023          └─ thread 1023: offset 2047

   All threads cooperate              All threads cooperate
   nvshmemx_float_put_block()         nvshmemx_float_put_block()
   (1024 elements)                    (1024 elements)

```

### 3. 集体启动 - Collective Launch

### 3.1 功能描述

演示`nvshmemx_collective_launch` API，确保所有PE同时启动相同的kernel，适用于需要全局同步的算法（如迭代式归约）。

### 3.2 源代码 (`collective-launch.cu`)

```
<span class="cp">#include</span> <span class="cpf">"nvshmemx.h"</span><span class="cp"></span>

<span class="c1">// Kernel: 环形归约算法</span>
<span class="n">__global__</span> <span class="kt">void</span> <span class="nf">reduce_ring</span><span class="p">(</span><span class="kt">int</span> <span class="o">*</span><span class="n">target</span><span class="p">,</span> <span class="kt">int</span> <span class="n">mype</span><span class="p">,</span> <span class="kt">int</span> <span class="n">npes</span><span class="p">)</span> <span class="p">{</span>
    <span class="kt">int</span> <span class="n">peer</span> <span class="o">=</span> <span class="p">(</span><span class="n">mype</span> <span class="o">+</span> <span class="mi">1</span><span class="p">)</span> <span class="o">%</span> <span class="n">npes</span><span class="p">;</span>
```

### 3.4 关键特性

| 特性 | 普通Launch | Collective Launch |

### 4. 信号同步 - Ring Broadcast

### 4.3 信号机制详解

```
<span></span>PE 0 (Root)        PE 1              PE 2              PE 3
  │                  │                 │                 │
  ├─ *psync = 1      │                 │                 │
  │  (数据已准备)     │                 │                 │
  │                  │                 │                 │
  ├─ put(data→PE1)   │                 │                 │
  ├─ fence           │                 │                 │
  ├─ signal_op       │                 │                 │
  │      ─────────→  ├─ wait_until     │                 │
  │                  │  (收到信号)      │                 │
  │                  ├─ put(data→PE2)  │                 │
  │                  ├─ fence          │                 │
  │                  ├─ signal_op      │                 │
  │                  │      ─────────→ ├─ wait_until    │
  │                  │                 │  (收到信号)     │
  │                  │                 ├─ put(data→PE3) │
  │                  │                 ├─ fence         │
  │                  │                 ├─ signal_op     │
  │                  │                 │      ────────→ ├─ wait_until
  │                  │                 │                │  (收到信号)
  │                  │                 │                └─ return (最后PE)

```

### 4.4 信号API说明

| API | 功能 | 参数 |
| --- | --- | --- |
| nvshmem_signal_wait_until(sig, cmp, val) | 等待信号满足条件 | sig地址, 比较操作, 比较值 |
| nvshmemx_signal_op(sig, val, op, pe) | 向远程PE发送信号 | sig地址, 值, 操作, 目标PE |
| nvshmem_fence() | 确保之前的RMA完成 | - |

**比较操作**:
- NVSHMEM_CMP_EQ: 等于
- NVSHMEM_CMP_NE: 不等于
- NVSHMEM_CMP_GT: 大于
- NVSHMEM_CMP_GE: 大于等于
- NVSHMEM_CMP_LT: 小于
- NVSHMEM_CMP_LE: 小于等于

**信号操作**:
- NVSHMEM_SIGNAL_SET: 设置为指定值

```
    <span class="n">correct_accumulate</span><span class="o"><<<</span><span class="mi">1</span><span class="p">,</span> <span class="n">input_nelems</span><span class="p">,</span> <span class="mi">0</span><span class="p">,</span> <span class="n">stream</span><span class="o">>>></span><span class="p">(</span><span class="n">input</span><span class="p">,</span> <span class="n">partial_sum</span><span class="p">,</span> <span class="n">full_sum</span><span class="p">);</span>

    <span class="c1">// 单一同步点</span>
    <span class="n">cudaStreamSynchronize</span><span class="p">(</span><span class="n">stream</span><span class="p">);</span>

    <span class="n">printf</span><span class="p">(</span><span class="s">"[%d of %d] run complete </span><span class="se">\n</span><span class="s">"</span><span class="p">,</span> <span class="n">mype</span><span class="p">,</span> <span class="n">npes</span><span class="p">);</span>

    <span class="n">cudaStreamDestroy</span><span class="p">(</span><span class="n">stream</span><span class="p">);</span>
    <span class="n">nvshmem_free</span><span class="p">(</span><span class="n">input</span><span class="p">);</span>
    <span class="n">nvshmem_free</span><span class="p">(</span><span class="n">partial_sum</span><span class="p">);</span>
    <span class="n">nvshmem_free</span><span class="p">(</span><span class="n">full_sum</span><span class="p">);</span>
    <span class="n">nvshmem_finalize</span><span class="p">();</span>
    <span class="k">return</span> <span class="mi">0</span><span class="p">;</span>
<span class="p">}</span>

```

### 7. 性能优化建议

### 7.1 选择正确的API

| 数据大小 | 推荐API | 原因 |
| --- | --- | --- |
| 1-8 字节 | nvshmem_TYPE_p() | 单元素操作，低延迟 |
| 64B - 4KB | nvshmem_TYPE_put() | 向量化传输 |
| >4KB (P2P) | nvshmemx_TYPE_put_block() | 多线程并行拷贝 |
| >4KB (RDMA) | nvshmemx_TYPE_put_block() | 合并消息，减少开销 |
| 集合操作 | nvshmemx_*_block/warp() | 线程协作，高吞吐 |

### 7.2 内存分配策略

```
<span></span><span class="c1">//  推荐: 使用nvshmem_malloc分配对称内存</span>
<span class="kt">float</span> <span class="o">*</span><span class="n">data</span> <span class="o">=</span> <span class="p">(</span><span class="kt">float</span><span class="o">*</span><span class="p">)</span><span class="n">nvshmem_malloc</span><span class="p">(</span><span class="n">size</span><span class="p">);</span>

<span class="c1">//  错误: 使用cudaMalloc的内存不能用于NVSHMEM</span>
<span class="kt">float</span> <span class="o">*</span><span class="n">data</span><span class="p">;</span>
<span class="n">cudaMalloc</span><span class="p">(</span><span class="o">&</span><span class="n">data</span><span class="p">,</span> <span class="n">size</span><span class="p">);</span>  <span class="c1">// 不能用于nvshmem_put/get</span>

<span class="c1">//  推荐: 对于非对称内存，使用buffer registration</span>
<span class="kt">void</span> <span class="o">*</span><span class="n">buf</span><span class="p">;</span>
<span class="n">cudaMalloc</span><span class="p">(</span><span class="o">&</span><span class="n">buf</span><span class="p">,</span> <span class="n">size</span><span class="p">);</span>
<span class="n">nvshmemx_buffer_register</span><span class="p">(</span><span class="n">buf</span><span class="p">,</span> <span class="n">size</span><span class="p">);</span>  <span class="c1">// NVSHMEM 2.0+</span>

```

### 7.3 同步优化

```
<span></span><span class="c1">//  低效: 过度同步</span>
<span class="k">for</span> <span class="p">(</span><span class="kt">int</span> <span class="n">i</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span> <span class="n">i</span> <span class="o"><</span> <span class="n">N</span><span class="p">;</span> <span class="n">i</span><span class="o">++</span><span class="p">)</span> <span class="p">{</span>
    <span class="n">nvshmem_int_put</span><span class="p">(...);</span>
    <span class="n">nvshmem_quiet</span><span class="p">();</span>  <span class="c1">// 每次都等待完成</span>
<span class="p">}</span>

<span class="c1">//  优化: 批量操作后统一同步</span>
<span class="k">for</span> <span class="p">(</span><span class="kt">int</span> <span class="n">i</span> <span class="o">=</span> <span class="mi">0</span><span class="p">;</span> <span class="n">i</span> <span class="o"><</span> <span class="n">N</span><span class="p">;</span> <span class="n">i</span><span class="o">++</span><span class="p">)</span> <span class="p">{</span>
    <span class="n">nvshmem_int_put_nbi</span><span class="p">(...);</span>  <span class="c1">// 非阻塞</span>
<span class="p">}</span>
<span class="n">nvshmem_quiet</span><span class="p">();</span>  <span class="c1">// 一次等待所有完成</span>

<span class="c1">//  最优: 使用fence分离顺序</span>
<span class="n">nvshmem_put</span><span class="p">(...);</span>
<span class="n">nvshmem_fence</span><span class="p">();</span>  <span class="c1">// 仅保证顺序，不等待完成</span>
<span class="n">nvshmemx_signal_op</span><span class="p">(...);</span>  <span class="c1">// 立即发送信号</span>
<span class="n">nvshmem_quiet</span><span class="p">();</span>  <span class="c1">// 最后统一等待</span>

```
