### 1. 逻辑视图 (Logical View)

逻辑视图描述系统的功能分解、核心抽象和模块之间的依赖关系。

### 1.1 核心架构层次

TorchComms 采用**四层架构**设计：

```
<span></span>┌─────────────────────────────────────────────────────────────┐

└────────────────────────┬────────────────────────────────────┘
                         │
┌────────────────────────▼────────────────────────────────────┐
│                 后端实现层 (Backend Layer)                    │
│  ┌──────────┬──────────┬──────────┬──────────┬──────────┐  │

```

### 1.2 核心模块功能分解

### 1.2.1 API 抽象层

**TorchComm 类** (`TorchComm.hpp`)
- 职责：统一的通信 API 门面
- 关键方法：
- 点对点通信：`send()`, `recv()`
- 集合通信：`all_reduce()`, `broadcast()`, `all_gather()`, `reduce_scatter()`, `all_to_all()`
- 分散收集：`scatter()`, `gather()`
- 通信器管理：`split()`, `finalize()`
- 窗口操作：`window_allocate()`

**TorchCommBackend 接口** (`TorchCommBackend.hpp`)
- 职责：后端实现的抽象接口
- 设计模式：策略模式（Strategy Pattern）
- ABI 版本控制：`TORCHCOMM_BACKEND_ABI_VERSION = "1.0"`
**TorchWork 类** (`TorchWork.hpp`)
- 职责：异步操作的句柄
- 功能：`is_completed()`, `wait()`, CUDA Graph 支持
### 1.2.2 通信传输层（CTran Framework）

**CTran 核心组件**：

```
<span></span>class CtranComm {

**代码示例**：

```
<span></span>import torch
from torchcomms import new_comm, ReduceOp
import time

comm = new_comm("ncclx", torch.device("cuda"))

# 创建通信和计算张量
comm_tensor = torch.randn(10 * 1024 * 1024, device="cuda")  # 40 MB
compute_tensor = torch.randn(1000, 1000, device="cuda")

# 异步启动通信
work = comm.all_reduce(comm_tensor, ReduceOp.SUM, async_op=True)

# 在通信进行的同时执行计算
for _ in range(100):
    compute_tensor = torch.mm(compute_tensor, compute_tensor.t())

# 等待通信完成
work.wait()

print("Communication and computation completed")

```

```

│                       ├─ Overlap: 10ms

    │             │<───────────┤            │
    │ work (graph │            │            │
    │<───────────────────────────┤            │
    │  mode)       │            │            │
    │             │            │            │
    │  [不调用 work.wait()]      │            │
    │             │            │            │
    │end_capture  │            │            │
    ├────────────>│            │            │
    │             │            │            │
    │  ┌─────────────────────────────┐
    │  │ Graph 重放阶段 (100x)     │
    │  └─────────────────────────────┘
    │ replay()     │            │            │
    ├────────────>│            │            │
    │             │Execute kernels         │
    │             ├───────────>│            │
    │             │            │            │
    │ ...         │            │            │
    │             │            │            │
    │synchronize  │            │            │
    ├────────────>│            │            │
    │             │            │            │

```

**代码示例**：

```
<span></span>import torch
from torchcomms import new_comm, ReduceOp

comm = new_comm("ncclx", torch.device("cuda"))
tensor = torch.randn(1024, device="cuda")

# 创建 CUDA Graph
graph = torch.cuda.CUDAGraph()

# 捕获通信操作
with torch.cuda.graph(graph):
    work = comm.all_reduce(tensor, ReduceOp.SUM, async_op=True)
    # 注意：不要在捕获阶段调用 work.wait()

# 重放 graph（优化的执行路径）
for _ in range(100):
    graph.replay()

# 同步
torch.cuda.current_stream().synchronize()

print(f"Result: {tensor[0].item()}")

```

**优势**：
- 消除 CPU 开销：每次迭代无需重复 API 调用
- 优化内核调度：CUDA Driver 优化 kernel 序列
- 降低延迟：减少 CPU-GPU 交互
**性能对比**：

```
<span></span>标准方式（无 Graph）：
- CPU overhead per iteration: ~200 μs
- 100 iterations: 20 ms CPU 时间

CUDA Graph 方式：
- CPU overhead (capture): ~500 μs (一次性)
- CPU overhead per replay: ~10 μs
- 100 iterations: 1 ms CPU 时间
- 加速比: 20x

```

<span></span>import torch
from torchcomms import new_comm, ReduceOp

# 创建全局通信器（8 个 rank）
global_comm = new_comm("ncclx", torch.device("cuda"), name="global")
rank = global_comm.get_rank()
world_size = global_comm.get_size()  # 8

# 创建子通信器（前 4 个 rank）
if rank < 4:
    sub_ranks = [0, 1, 2, 3]
    sub_comm = global_comm.split(sub_ranks, name="group_0")

    # 在子组内执行 AllReduce
    tensor = torch.ones(1024, device="cuda") * (rank + 1)
    sub_comm.all_reduce(tensor, ReduceOp.SUM, async_op=False)

    print(f"Rank {rank} sub-group result: {tensor[0].item()}")
    # 输出：10.0 (1+2+3+4)

    sub_comm.finalize()

global_comm.finalize()

```
