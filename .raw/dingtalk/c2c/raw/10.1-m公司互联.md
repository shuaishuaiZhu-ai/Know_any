采用LD/ST方式，类似NV方案。
1.
硬件实现
a.
serdes + pcs/ilkn + axi controller/bridge + noc做法来进行互联，不需要主从关系，对等的training
b.
互连之间有边带信号连接，做link握手
c.
互联之间要有中断实现
d.
pcie gpu和 port连接NOC之间放一个地址转换单元AMT，解析global mem地址，指向那一个GPU，该从那个互连接口出去
i.
AMT需要软件根据top路由配置地址路由
e.
支持40 bit地址，1T地址空间
f.
top也是固定几种连接方式（理论上没有限制，软件能够通过port link关系，识别任何top关系）
2.
fw
a.
link traing
i.
serdes link training
ii.
pcs/ilkn link training
b.
port和chip关系表建立
c.
配合kmd进行AMT配置，其实就是AMT地址路由表配置
d.
互联的port之间，trigger msg通讯，交换port号和chip id号，形成一张地址路由表
e.
异常处理
3.
kmd
a.
kmd会预先定制各种top互联关系
b.
根据pcie bdf号配置chip id
c.
根据port和chip号，确认top互联关系
d.
找出chip和chip之间最优路径
e.
联动fw配置AMT路由
f.
异常处理

AMT路由配置流程：
1.
fw先做好link，插在服务器上的卡pcie也link
2.
kmd拿到各个卡的bdf信息，通过一定规则，指定GPU ID-chip id，并下发给FW
3.
FW收到GPU ID（chip0-3）之后，FW开始配置寄存器，以广播形式，让link的port之间互相发自己的port信息和chip id信息，确认各个chip的互联port和port之间关系路径，FW收集所有不成环的路径并保存
4.
以GPU0发起点举例，流程如下：
a.
FW检查GPU0 port0/1/2状态是link，通过构造source是chip 0/port0，chip 0/port1，chip 0/port2，分别填入port0/1/2对应的寄存器，trigger中断 msg携带source信息，发送到对方link的port
b.
GPU1的port1收到中断，解析msg信息写入寄存器，GPU1 FW保存并记录，GPU1同样构造chip1 port 0为source， chip0 port 0为dest，发送给chip0，作为一个ACK
c.
GPU0收到后，在重复发一次source为chip 0 port 0 dest为chip1 port 0，re-ACK一次
d.
这样GPU0和GPU1间建立了一项通讯表项：
i.
GPU 0 - source：chip 0 port 0 dest：chip1 port 0
ii.
GPU 1 - source：chip 1 port 0 dest：chip0 port 0
e.
GPU2 GPU3和GPU0也是同样通讯
f.
最终GPU0会有最优表项：
i.
source：chip 0 port 0 dest：chip1 port 0
ii.
source：chip 0 port 1 dest：chip2 port 1
iii.
source：chip 0 port 2 dest：chip3 port 2
g.
GPU1会有最优表项：
i.
source：chip 1 port 0 dest：chip0 port 0
ii.
source：chip 1 port 1 dest：chip3 port 1
iii.
source：chip 1 port 2 dest：chip2 port 2
h.
GPU2会有最优表项：
i.
source：chip 2 port 0 dest：chip3 port 0
ii.
source：chip 2 port 1 dest：chip0 port 1
iii.
source：chip 2 port 2 dest：chip1 port 2
i.
GPU3会有最优表项：
i.
source：chip 3 port 0 dest：chip2 port 0
ii.
source：chip 3 port 1 dest：chip1 port 1
iii.
source：chip 3 port 2 dest：chip0 port 2
j.
表项中source和dest的chip id相等，认为是死锁情况，互联接口自己插在自己port上了，不允许。
