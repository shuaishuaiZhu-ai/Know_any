其中：
●

●

●
TPID (Tag Protocol Identifier， 标签协议标识符)，长度：2字节
○
作用：相当于一个“标志位”，告诉交换机或网卡后面跟着的是VLAN信息，不是普通的长度/类型字段。
○
固定值：0x8100。
●
TCI (Tag Control Information， 标签控制信息)，长度：2字节，这16个比特位又被细分为三个部分：
○
PCP (Priority Code Point， 优先级代码点)：3比特
■
取值范围：0（最低）到 7（最高）。
■
作用：用于实现 QoS，标识帧的优先级。当网络拥塞时，高优先级的数据会被优先转发。
○
DEI (Drop Eligible Indicator， 丢弃合格指示符)：1比特，以前标准中称为 CFI。
■
作用：用于拥塞控制。当网络发生拥塞时，DEI=1 的帧会被优先考虑丢弃。
●
VID (VLAN Identifier， VLAN 标识符)：12比特这是 VLAN 划分的核心。
○
取值范围：0 到 4095。其中，0 和 4095 是保留值，所以实际可用的 VLAN ID 范围是 1 到 4094。
○
作用：标识该数据帧属于哪一个具体的 VLAN。

2.
交换机转发原理
2.1 标准L2报文转发原理
参考H3C和盛科L2交换机，支持静态MAC，动态MAC和黑洞MAC，他们区别：
●
静态MAC，需要用户自己配置，除了用户，不会更改，也不老化，不自动更新
●
动态MAC，也可以称为MAC学习，需要相互之间L2报文转发，进行MAC学习，有老化和更新流程
●
黑洞MAC，是为了用于丢弃含有特定目的MAC 地址的报文，作为安全考虑
举例如下：
●
静态MAC配置或者动态学习：
●


○
device0和device3在同一个vlan域（不同vlan域之间报文不能转发，比如device0想转发报文到device1是不可以的，会被drop掉）
○
静态路由表由用户按照topo情况，直接配置就行
○
动态MAC表学习过程：
■
device给交换机发携带自己SMAC和vlan的报文，交换机收到了报文，解析出SMAC，vlan id和记录收到报文端口，并将这些信息写到MAC表
