## Errors Encountered
- **sed multi-line append through SSH**: Using `/pattern/a\` with multiple lines corrupted `drv_dms.h` with literal sed command text. Reverted with `git checkout`. Root cause: sed multi-line syntax does not survive the SSH + shell escaping pipeline.
- **Python heredoc content matching failure**: Trailing whitespace differences between expected and actual file content caused exact-string matching to fail silently. Diagnosed with `cat -A`.
- **Backslash-n expansion through SSH heredoc layers**: A `\n` inside a printf format string was expanded to a literal newline by the shell/heredoc before Python received it. Five consecutive approaches failed (Python heredoc, Python one-liner, sed, base64-encoded Python, and direct string). Only the `chr(92)+chr(110)` approach survived all escaping layers.

## Patterns Used
- SSH `cat -n` for reading remote files (established pattern from prior sessions)
- Python via SSH heredoc for bulk file edits (works for most content)
- `chr()` construction in Python to bypass all shell escape interpretation (new pattern)
- Line-number-based replacement instead of content matching (more robust against invisible whitespace)
- `git checkout` for immediate recovery from corrupted edits
- `cat -A` to reveal trailing whitespace and line endings when debugging match failures
