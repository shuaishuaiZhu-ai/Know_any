## Root Cause Found

**Copilot Chat 0.43.0 (installed 2026-04-08 17:23) added `"plan"` to the special agent type array, causing VS Code's built-in Plan Agent to hijack plan mode requests before they reach the Claude Code CLI.**

### Evidence

- `extension.js` line 2250 (0.42.3): `qze=["compact"]` — only compact is a special agent
- `extension.js` line 2372 (0.43.0): `bke=["compact","plan","fleet"]` — **plan and fleet added**
- Both versions' `cli.js` contain identical `Plan mode is active` injection code at `case "plan_mode"` — the CLI code is fine
- The break is in the routing: 0.43.0's extension intercepts "plan" requests and routes them to its own Plan Agent (prompt at lines 1413-1497, uses `/memories/session/plan.md`), bypassing the CLI's plan mode system entirely
- CLI never receives plan mode state → never injects `Plan mode is active` system reminder → Claude can't detect plan mode
