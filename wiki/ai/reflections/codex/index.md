# Codex 反思与进化

这里保存所有 Codex 自我反思、项目复盘、automation 进化和工作流改进文档。

## 目录

| 分类 | 路径 | 用途 |
|---|---|---|
| 每日自我反思 | [daily](<./daily/>) | Daily Codex Self-Review 生成的每日复盘。 |
| 项目自我反思 | [projects](<./projects/>) | 用户要求对某个项目、任务、会话做总结反思时生成的文档。 |
| 进化文档 | [evolution](<./evolution/>) | automation、prompt、skill、工作流规则的改进记录。 |

## 写入规则

- 只要用户要求“自我反思”、“总结反思”、“复盘”、“项目总结”、“进化建议”，都要在本目录下生成对应 Markdown 文档。
- 每日 automation 仍保留项目内 `notes/codex-daily/YYYY-MM-DD.md`，同时在 `daily/YYYY-MM-DD.md` 写入 Obsidian 版本。
- 项目或专题复盘使用 `projects/YYYY-MM-DD-主题.md`。
- automation、prompt、skill 的演进使用 `evolution/YYYY-MM-DD-主题.md`。
- 文档必须写清证据、结论、未验证项、下一步，不把推测写成事实。
- 涉及 token、账号、密钥、私密路径时只写“发现疑似敏感信息，已省略具体值”。
